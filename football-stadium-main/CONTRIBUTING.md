# Contributing

Thank you for improving StadiView.

## Development

1. Fork the repository.
2. Create a focused branch from the default branch.
3. Install dependencies with `npm install`.
4. Run the app with `npm start`.
5. Verify a production build with `npm run build`.
6. Open a pull request describing the change and how it was tested.

Keep changes focused, preserve existing stadium interactions, and avoid
committing generated output from `dist/`, Angular cache files, or dependencies.
